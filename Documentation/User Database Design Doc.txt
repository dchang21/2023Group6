User Database Design Document
Last Updated: 10/13/2023

NOTE:
Updated version of a mock user database design from product proposal.
This design is a representation of what the user database should be for our website EurekaEats
Design will be updated to meet new changes 

Mongodb items in database:
Field id - unique id assigned to every user account. Type int
Field username -  username of the user. Type String
Field first_name - first name of the user. Type String
Field last_name - last name of the user. Type String
Field location - general coordinate of the user. Use google location feature to obtain the values. Type [double,double]  
Field eating_preferences - eating preferences of the user. Type Object
  Field vegan - is the user vegan. Type Boolean
  Field keto - do the user prefer keto. Type Boolean
  Field gluten_free : do the user prefer gluten free. Type Boolean
	Field protein - what type of protein they prefer. Type Object
    Field beef - users prefer beef. Type Boolean
    Field chicken - users prefer chicken. Type Boolean
    Field pork - users prefer pork. Type Boolean
    Field tofu - users prefer tofu/soy. Type Boolean
    Field lamb - users prefer lamb. Type Boolean
  Field cuisine - what type of cuisine they prefer. Type Object
    Field Italian - prefer italian food. Type boolean
    Field Chinese - prefer Chinese food. Type boolean
Field reviews - user reviews of a restaurant. Type Object
  Field restaurant_id - restaurant id for access to the restaurant database. Type int
  Field restaurant_name - the restaurant name. Type String
  Field star_rating - the star rating review of the restaurant. Type int
  Field comment - the user comment for the restaurant. Type String
